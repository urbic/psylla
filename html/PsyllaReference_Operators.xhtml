5.2.76. eq

  • result: boolean

The meaning of object equality depends on their types. Objects of the same simple type are considered equal if their values are the same. Objects of numeric types are considered equal if they represent the same mathematical number. The objects of the string type are considered equal if their string values are the same. Objects of other complex types are equal only if they are identical.

It should be noted that in mathematical calculations, there may be loss of precision due to rounding, so that the expected equality may be violated.

	▶ true	# Equal real and integer numbers

	▶ true	# Name and string with equal content ane equal
[ 0 1 2 ] dup eq
	▶ true	# Two instances of the same array are equal
[ 0 1 2 ] [ 0 1 2 ] eq
	▶ false	# Two distinct instances of arrays are not equal
1 77777777 div 77777777 mul 1 eq
	▶ false	# Equality is broken due to the loss of accuracy
