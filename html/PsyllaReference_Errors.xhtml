6.1. Errors in Detail

6.1.1. concurrentmodification

Concurrent modification detected.

6.1.2. dictstackunderflow

Dictionary stack underflow. The error is generated by the end operator, when the dictionary stack contains only permanent dictionaries.

6.1.4. directorynotempty

The directory is not empty. The error is generated by the operators expecting that the directory is empty, when this condition fails.

6.1.5. fileaccessdenied

File access denied.

6.1.6. fileexists

File exists. The error is generated by the operators expecting that the file does not exist, when this condition fails.

6.1.7. filenotfound

File not found. The error is generated by the operators expecting that the file exists, when this condition fails.

6.1.9. invalidcontext

Invalid context.

6.1.10. invalidexit

Invalid exit. The error is generated by the exit operator when invoked outside the cyclic context.

6.1.11. invalidexternal

TODO

6.1.12. invalidregexp

Invalid regular expression. The error is generated by the parser or by the regexp operator when the regular expression is syntactically inconsistent.

6.1.13. invalidstate

Invalid state. The error is generated if the execution of the operator is not possible with the current state of the operand or environment.

6.1.14. ioerror

Input/output error.

6.1.15. limitcheck

System limits exceeded.

6.1.16. notdirectory

Not a directory.

6.1.17. notimplemented

Unimplemented operation. The error is generated by the operators trying to perform an unimplemented operation.

6.1.18. notlink

Not a link.

6.1.19. rangecheck

The error is generated when the operand has the value outside the range acceptable for the operator.

6.1.20. securityerror

Security policy violation.

6.1.21. stackunderflow

Operand stack underflow. The error is generated when the operand stack contains fewer operands than the operator expects.

6.1.22. syntaxerror

Syntax error.

6.1.23. typecheck

Type mismatch. The error is generated if the operand has the type that is unexpected for the operator.

6.1.24. undefined

Undefined key. The error is generated by the operator accessing the dictionary by the absent key. Also, is generated by the interpreter failing to resolve the command name against the dictionary stack.

6.1.25. undefinedresult

Undefined result. The error is generated when the result of the operation is not defined, for example, when trying to integer division by zero in the idiv operator.

6.1.26. unmatchedmark

Unmatched mark. The error is generated by the operators expecting the mark-delimited list of operands when the mark is absent in the operand stack.

6.1.27. unsupported

Unsupported operation. The error is generated by the operators trying to perform an unsupported operation.
